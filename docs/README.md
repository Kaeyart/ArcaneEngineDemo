## Product and technical records

Existing patch notes, implementation manifests, reaction codices, and validation checklists remain authoritative
records for their versions.

Target organization for future releases:

```text
docs/
  architecture/
  design/
  development/
  releases/
    2.x/
    3.x/
  validation/
```

The first repository cleanup intentionally does not move historical documents. Moving them should be a separate
commit with link verification so that history remains reviewable.
