# Security Policy

## Supported versions

Arcane Engine Demo is an active prototype. Security fixes are considered for the current maintenance line and the
active 3.0 development line.

## Reporting a vulnerability

Do not publish secrets, private data, exploit details, or sensitive save files in a public issue.

Use GitHub private vulnerability reporting or a private security advisory when enabled. When that channel is not
available, open a minimal public issue requesting a private contact channel without including sensitive details.

Include:

- Affected version or commit.
- Platform and Unity version.
- Reproduction steps.
- Impact.
- Relevant logs with credentials and personal paths removed.
